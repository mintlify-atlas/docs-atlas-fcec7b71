This guide walks you through loading and playing a Lottie animation from your app bundle. Choose the tab that matches your UI framework.

<Steps>
  <Step title="Install Lottie">
    Follow the [Installation](/installation) guide to add Lottie to your project, then import it:

    ```swift
    import Lottie
    ```
  </Step>

  <Step title="Add an animation file to your project">
    Drag a `.json` (Bodymovin) or `.lottie` (DotLottie) animation file into your Xcode project. Make sure **Copy items if needed** is checked and the file is added to your app target so it ends up in the main bundle.

    <Tip>
      Free animations are available at [LottieFiles](https://lottiefiles.com). Download any animation as a `.json` file and drop it straight into your Xcode project.
    </Tip>
  </Step>

  <Step title="Display the animation">
    Pick the approach for your UI framework below.

    <Tabs>
      <Tab title="UIKit">
        `LottieAnimationView` is a `UIView` subclass decorated with `@IBDesignable`, so you can use it in code or in Interface Builder.

        ```swift
        import UIKit
        import Lottie

        class ViewController: UIViewController {

            override func viewDidLoad() {
                super.viewDidLoad()

                // 1. Load the animation from the main bundle by file name (no extension needed).
                //    Returns nil if the file is not found — handle that case in production code.
                guard let animation = LottieAnimation.named("MyAnimation") else { return }

                // 2. Create the animation view with the loaded animation.
                let animationView = LottieAnimationView(animation: animation)
                animationView.translatesAutoresizingMaskIntoConstraints = false

                // 3. Configure playback behavior.
                animationView.loopMode = .loop      // loop continuously
                animationView.animationSpeed = 1.0  // normal speed

                // 4. Add to the view hierarchy and constrain it.
                view.addSubview(animationView)
                NSLayoutConstraint.activate([
                    animationView.centerXAnchor.constraint(equalTo: view.centerXAnchor),
                    animationView.centerYAnchor.constraint(equalTo: view.centerYAnchor),
                    animationView.widthAnchor.constraint(equalToConstant: 300),
                    animationView.heightAnchor.constraint(equalToConstant: 300),
                ])

                // 5. Start playback.
                animationView.play { finished in
                    // Called when the animation stops.
                    // `finished` is false if the animation was interrupted.
                    print("Animation finished: \(finished)")
                }
            }
        }
        ```

        <Note>
          `LottieAnimation.named(_:bundle:subdirectory:animationCache:)` looks for the file in `Bundle.main` by default and caches the decoded model in `LottieAnimationCache.shared`. Subsequent calls with the same name return the cached animation with no disk I/O.
        </Note>

        **Playing between specific progress values:**

        ```swift
        // Play from 0% to 50% once, then call the completion block.
        animationView.play(fromProgress: 0, toProgress: 0.5, loopMode: .playOnce) { _ in
            print("First half done")
        }
        ```

        **Scrubbing manually:**

        ```swift
        // Jump to the midpoint without playing.
        animationView.currentProgress = 0.5
        ```
      </Tab>

      <Tab title="SwiftUI">
        `LottieView` is the SwiftUI wrapper around `LottieAnimationView`. The async closure initializer loads the animation the first time the view appears, keeping your UI responsive.

        ```swift
        import SwiftUI
        import Lottie

        struct ContentView: View {
            var body: some View {
                LottieView {
                    // This closure runs asynchronously on first appear.
                    // LottieAnimation.named(_:) is synchronous but inexpensive for
                    // cached or small files. Wrap in Task.detached for large files.
                    LottieAnimation.named("MyAnimation")
                } placeholder: {
                    ProgressView()
                }
                .playing(loopMode: .loop)  // loop continuously
                .resizable()              // scale to fill the offered space
                .frame(width: 300, height: 300)
            }
        }
        ```

        If you have already loaded a `LottieAnimation` synchronously (e.g., at app launch), pass it directly without the async closure:

        ```swift
        struct ContentView: View {
            let animation: LottieAnimation?

            var body: some View {
                LottieView(animation: animation)
                    .playing(loopMode: .loop)
                    .resizable()
                    .frame(width: 300, height: 300)
            }
        }
        ```

        **Common `LottieView` modifiers:**

        | Modifier | Description |
        |----------|-------------|
        | `.playing()` | Play once from the current frame to the end. |
        | `.playing(loopMode:)` | Play with the specified `LottieLoopMode`. |
        | `.looping()` | Convenience — loop continuously from start to end. |
        | `.paused(at:)` | Pause at a specific frame or progress. |
        | `.resizable()` | Scale the animation to fill the proposed size. |
        | `.intrinsicSize()` | Use the animation's natural size, up to the proposed size. |
        | `.animationSpeed(_:)` | Multiply the playback speed. |

        <Note>
          The `LottieView` async closure is called exactly once in `onAppear`. To reload the animation at a different time (e.g., after a network fetch), use the `.reloadAnimationTrigger(_:)` modifier.
        </Note>
      </Tab>
    </Tabs>
  </Step>
</Steps>

## Loop modes

Control repetition with `LottieLoopMode`, used by both `LottieAnimationView` and `LottieView`:

```swift
animationView.loopMode = .playOnce           // play once and stop (default)
animationView.loopMode = .loop               // loop from start to end indefinitely
animationView.loopMode = .autoReverse        // play forward then backward, indefinitely
animationView.loopMode = .repeat(3)          // loop exactly 3 times
animationView.loopMode = .repeatBackwards(2) // forward + backward, 2 times
```

## Choosing a rendering engine

Lottie defaults to the Core Animation engine, which has zero CPU overhead during playback. Switch engines via `LottieConfiguration`:

```swift
// Core Animation (default) — runs on the render server, zero CPU overhead.
let animationView = LottieAnimationView(
    animation: animation,
    configuration: LottieConfiguration(renderingEngine: .coreAnimation)
)

// Main Thread — required for features Core Animation cannot express.
let animationView = LottieAnimationView(
    animation: animation,
    configuration: LottieConfiguration(renderingEngine: .mainThread)
)
```

Set the global default so every new `LottieAnimationView` inherits it:

```swift
LottieConfiguration.shared = LottieConfiguration(renderingEngine: .coreAnimation)
```

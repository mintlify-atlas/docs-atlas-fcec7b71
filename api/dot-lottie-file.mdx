title: "DotLottieFile"
description: "Load and work with .lottie compressed animation bundles"
---

`DotLottieFile` represents a parsed `.lottie` archive. It exposes one or more named `LottieAnimation` objects together with their per-animation playback configuration (`DotLottieConfiguration`).

## What is the .lottie format?

A `.lottie` file is a ZIP archive that may contain:

- One or more animation JSON files (in an `animations/` or `a/` directory)
- Shared image assets (in an `images/` or `i/` directory)
- A `manifest.json` that describes each animation's ID and playback presets

<Note>
  `.lottie` file support was introduced in Lottie 4.x. Each archive can hold multiple named animations with individual loop modes and playback speeds, all sharing a single image asset folder.
</Note>

## Loading methods

All loading methods are `static` on `DotLottieFile`. Prefer the `async throws` variants in modern Swift concurrency contexts; the callback-based overloads are available for Objective-C compatibility or older codebases.

<Warning>
  The methods inside `DotLottieFile.SynchronouslyBlockingCurrentThread` block the calling thread and **must not** be called on the main thread. Use the `async` or callback variants instead.
</Warning>

### Load by name from bundle

```swift
public static func named(
  _ name: String,
  bundle: Bundle = Bundle.main,
  subdirectory: String? = nil,
  dotLottieCache: DotLottieCacheProvider? = DotLottieCache.sharedCache
) async throws -> DotLottieFile
```

<ParamField path="name" type="String" required>
  The filename without the `.lottie` extension. For example, `"StarAnimation"` loads `StarAnimation.lottie`.
</ParamField>

<ParamField path="bundle" type="Bundle" default="Bundle.main">
  The bundle that contains the `.lottie` file.
</ParamField>

<ParamField path="subdirectory" type="String" default="nil">
  An optional subdirectory within the bundle where the file is located.
</ParamField>

<ParamField path="dotLottieCache" type="DotLottieCacheProvider?" default="DotLottieCache.sharedCache">
  A cache used to avoid re-parsing the same file. Pass `nil` to bypass caching.
</ParamField>

A callback-based overload is also available for contexts that cannot use Swift concurrency:

```swift
public static func named(
  _ name: String,
  bundle: Bundle = Bundle.main,
  subdirectory: String? = nil,
  dotLottieCache: DotLottieCacheProvider? = DotLottieCache.sharedCache,
  dispatchQueue: DispatchQueue = .dotLottie,
  handleResult: @escaping (Result<DotLottieFile, Error>) -> Void
)
```

## DotLottieConfiguration

Playback presets extracted from the `.lottie` manifest for a single animation.

```swift
public struct DotLottieConfiguration
```

<ParamField path="id" type="String" required>
  The animation's unique identifier as declared in the manifest.
</ParamField>

<ParamField path="loopMode" type="LottieLoopMode">
  The loop behavior specified in the manifest (for example `.loop`, `.playOnce`).
</ParamField>

<ParamField path="speed" type="Double">
  Playback speed multiplier from the manifest. `1.0` means normal speed.
</ParamField>

<ParamField path="imageProvider" type="AnimationImageProvider?">
  An image provider pre-configured to load assets bundled inside the `.lottie` archive. This is set automatically when the archive contains an `images/` directory.
</ParamField>

### DotLottieConfigurationComponents

An `OptionSet` that controls which parts of a `DotLottieConfiguration` are applied when the animation is loaded into a `LottieView` or `LottieAnimationView`.

```swift
public struct DotLottieConfigurationComponents: OptionSet {
  public static let imageProvider    // apply the bundled image provider
  public static let loopMode         // apply the manifest loop mode
  public static let animationSpeed   // apply the manifest playback speed
  public static let all              // apply all components
  public static let none             // apply no components
}
```

<Tip>
  When using `LottieView`, call `.dotLottieConfigurationComponents(.all)` to let the manifest fully control loop mode, speed, and images. Use `.dotLottieConfigurationComponents(.none)` if you want to override everything from SwiftUI modifiers.
</Tip>

## DotLottieCacheProvider

A protocol for implementing custom `.lottie` file caches.

```swift
public protocol DotLottieCacheProvider: Sendable {
  func file(forKey: String) -> DotLottieFile?
  func setFile(_ lottie: DotLottieFile, forKey: String)
  func clearCache()
}
```

### DotLottieCache

The built-in LRU cache shipped with Lottie. Stores up to 100 `DotLottieFile` objects by default. Once the limit is reached, the least-recently-used entry is evicted.

```swift
public final class DotLottieCache: DotLottieCacheProvider {
  public static let sharedCache: DotLottieCache

  /// Maximum number of cached files. Default is 100.
  public var cacheSize: Int

  public func clearCache()
}
```

<ParamField path="DotLottieCache.sharedCache" type="DotLottieCache">
  The global shared cache instance used by all loading methods unless overridden.
</ParamField>

<ParamField path="cacheSize" type="Int" default="100">
  The maximum number of `DotLottieFile` objects held in memory. Increasing this value trades memory for faster repeated loads.
</ParamField>

To bypass caching entirely, pass `nil` for `dotLottieCache` in any loading method.

---

## DispatchQueue.dotLottie

A serial `DispatchQueue` at `.userInitiated` quality-of-service used internally for all `.lottie` file I/O. Its serial nature prevents concurrent decompression of the same archive, which can cause file-system errors.

```swift
public static let dotLottie = DispatchQueue(
  label: "com.airbnb.lottie.dot-lottie",
  qos: .userInitiated
)
```

## Examples

<Steps>
  <Step title="Load a bundled .lottie file in SwiftUI">
    Use `LottieView`'s async initializer to load a `.lottie` file by name from the main bundle:

    ```swift
    import Lottie
    import SwiftUI

    struct AnimationView: View {
      var body: some View {
        LottieView {
          try await DotLottieFile.named("confetti")
        }
        .playing()
      }
    }
    ```
  </Step>
  <Step title="Load a .lottie file in UIKit">
    Await the file on a background context, then create a `LottieAnimationView`:

    ```swift
    import Lottie

    class MyViewController: UIViewController {
      override func viewDidLoad() {
        super.viewDidLoad()

        Task {
          let dotLottie = try await DotLottieFile.named("confetti")
          let animationView = LottieAnimationView(dotLottie: dotLottie)
          animationView.frame = view.bounds
          animationView.contentMode = .scaleAspectFit
          view.addSubview(animationView)
          animationView.play()
        }
      }
    }
    ```
  </Step>
  <Step title="Load from a remote URL">
    ```swift
    let url = URL(string: "https://example.com/animations/fireworks.lottie")!
    let dotLottie = try await DotLottieFile.loadedFrom(url: url)

    let animationView = LottieAnimationView(dotLottie: dotLottie)
    ```
  </Step>
  <Step title="Select a specific animation by ID">
    A `.lottie` bundle may contain multiple animations. Use `animations` to access them:

    ```swift
    let dotLottie = try await DotLottieFile.named("multi-state")

    // Play the animation with id "success"
    if let entry = dotLottie.animations.first(where: { $0.configuration.id == "success" }) {
      let animationView = LottieAnimationView(animation: entry.animation)
      animationView.loopMode = entry.configuration.loopMode
      animationView.animationSpeed = entry.configuration.speed
      animationView.play()
    }
    ```
  </Step>
  <Step title="Use a custom cache">
    Provide a custom `DotLottieCacheProvider` to integrate with your own memory management:

    ```swift
    struct NoCacheDotLottie: DotLottieCacheProvider {
      func file(forKey key: String) -> DotLottieFile? { nil }
      func setFile(_ lottie: DotLottieFile, forKey key: String) {}
      func clearCache() {}
    }

    let dotLottie = try await DotLottieFile.named(
      "confetti",
      dotLottieCache: NoCacheDotLottie()
    )
    ```
  </Step>
</Steps>
